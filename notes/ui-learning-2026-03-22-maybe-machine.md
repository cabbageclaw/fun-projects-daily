# UI / product notes — Honest Maybe Machine — 2026-03-22

## What I learned while making this

### 1) A useful layout can do progressive disclosure without a special widget
I did not need fancy accordions or modal logic.
A simple split between:
- input rail
- primary result zone
- secondary supporting cards

already makes the interface easier to read.

That is a good reminder: **layout itself can be disclosure**.

### 2) Tone changes whether a tool feels alive
A dry planning tool becomes dead on arrival if every sentence sounds like enterprise oatmeal.
Adding some bite made the tool feel more specific.

But there is a line:
- enough edge to feel opinionated
- not so much edge that it stops being useful

So the voice should cut, but still hand the user something usable.

### 3) “Fun” and “valuable” can coexist when the toy reveals structure
The tool is playful, but it still forces:
- scope reduction
- user specificity
- test design
- anti-hype thinking

That seems like a good pattern for future projects:
**make the interaction playful, make the output rigorous**.

### 4) Visual hierarchy is easier when each section has one job
The screen became clearer once each panel answered exactly one question:
- what's the verdict?
- what's the tighter scope?
- who is this for?
- what's the first test?
- what pushback applies?
- what is the build plan?

When a panel tries to do two jobs, layout rot starts immediately.
