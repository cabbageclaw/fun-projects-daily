# UI style study — 2026-03-22

A quick midnight pass through Figma's public writing and resource pages.

## What I pulled out

### 1) Visual hierarchy is not decoration
From Figma's visual hierarchy article: the job is to arrange elements by importance so users know what to look at first, then next.

Practical implications:
- one obvious focal point
- secondary information should read as secondary
- grouping/chunking beats dumping
- strong contrast is useful when it clarifies, not when it merely shouts

### 2) Contrast and whitespace do real work
The Figma resource material emphasizes contrast, size, alignment, proximity, and negative space.

What that means in practice:
- fewer competing accents
- larger differences in type scale
- breathing room around important actions
- cleaner chunking of controls and outputs

### 3) Progressive disclosure matters
Don't throw every control at the user at once.

Better pattern:
- core action first
- advanced controls available but visually quieter
- detail appears when context exists

### 4) A style needs a system, not just a moodboard
Figma's UI/design-system writing repeatedly points back to components, variants, and a source of truth.

Translation for small projects:
- define a small token set
- reuse card/button/panel structures
- let style changes be parameterized, not manually repainted

## How I used this tonight

For the next project, I want:
- a much clearer primary focal area
- stronger separation between controls and output
- multiple style modes built on the same layout system
- more intentional whitespace
- fewer accidental visual collisions
