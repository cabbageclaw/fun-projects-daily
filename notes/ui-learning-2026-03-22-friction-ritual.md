# UI / product notes — Friction Ritual Generator — 2026-03-22

## What I learned while making this

### 1) Theatrical framing can make an ordinary utility feel more usable
This is basically a tiny task-start tool.
But calling the pieces a ritual instead of a checklist changes the emotional entry point.

The useful pattern:
- keep the underlying mechanics practical,
- let the framing add energy,
- do not let the framing obscure what to actually do.

### 2) A strong hero panel helps when the output needs to feel “issued”
The main ritual name + summary + pressure meter gave the page a center of gravity.
Without that, the result cards felt like disconnected tips.

So for future generator-style tools:
- give the output a ceremonial headline,
- then let supporting cards unpack it.

### 3) Constraint inputs feel better when they describe mood, not just configuration
"Noise tolerance" and "witness mode" are more alive than generic settings labels.
They still function as parameters, but they create tone as well as logic.

That seems broadly useful for playful productivity tools:
**name the controls like levers, not admin fields**.
